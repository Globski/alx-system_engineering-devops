# System Engineering DevOps Projects

This repository contains projects focused on system engineering and DevOps practices, covering a variety of topics essential for managing and deploying applications effectively.

## Repository Structure

| Directory/Project | Description |
|-------------------|-------------|
| [0x00-shell_basics](./0x00-shell_basics) | Covers basic shell commands for file navigation, manipulation, and working with the command line. Tasks include printing the current directory, listing contents, creating and removing directories, and manipulating files. |
| [0x01-shell_permissions](./0x01-shell_permissions) | Focuses on managing file permissions in Unix/Linux systems. Tasks include changing user ownership, modifying file permissions for users and groups, creating files and directories with specific permissions, and using symbolic links. |
| [0x02-shell_redirections](./0x02-shell_redirections) | Focuses on input and output redirection in shell scripting. Tasks include printing messages, displaying file contents, manipulating text data, counting files, and performing various text processing operations. |
| [0x03-shell_variables_expansions](./0x03-shell_variables_expansions) | Focuses on shell variables and expansions. Tasks include creating and manipulating local and global variables, performing arithmetic operations, converting number bases, and handling environment variables. |
| [0x04-loops_conditions_and_parsing](./0x04-loops_conditions_and_parsing) | Focuses on fundamental Bash scripting concepts, including loops (for, while, until) and conditional statements (if, elif, else, case). The project includes tasks like generating SSH keys, implementing FizzBuzz, and parsing Apache logs. |
| [0x05-processes_and_signals](./0x05-processes_and_signals) | Focuses on processes and signals in shell scripting. It covers concepts like obtaining PIDs, managing processes, handling signals, and creating zombie processes in C. Tasks include displaying the PID, managing background processes, and signal response. |
| [0x06-regular_expressions](./0x06-regular_expressions) | Focuses on mastering regular expressions using the Oniguruma library in Ruby. The tasks involve creating regex patterns to match specific criteria, such as words, repetitions, phone numbers, and capital letters. This project enhances skills in pattern matching and string manipulation. |
| [0x07-networking_basics](./0x07-networking_basics) | This project covers the fundamentals of networking, including the OSI model, types of networks (LAN, WAN, Internet), MAC and IP addresses, TCP/UDP protocols, and more. It aims to provide insights into data transmission and device communication across networks. |
| [0x08-networking_basics_2](./0x08-networking_basics_2) | This project continues exploring networking basics, focusing on localhost, IP addressing, and network interfaces. It includes tasks to modify local network configurations, display active IPs, and create a listening port on localhost, enhancing your skills in configuring network settings and troubleshooting connectivity issues. |
| [0x09-web_infrastructure_design](./0x09-web_infrastructure_design) | This project explores the design of various web infrastructure setups, from a simple single-server configuration to a complex, secured, and monitored multi-server architecture. Tasks involve creating diagrams and explanations of each design, focusing on components like DNS, load balancers, firewalls, and database clusters to understand their roles in building scalable and reliable web applications. |
| [0x0A-configuration_management](./0x0A-configuration_management) | This project focuses on basic configuration management using Puppet. It includes tasks to create a file, install a package, and execute a command. The aim is to provide foundational knowledge in managing system configurations effectively through Puppet manifests. |
| [0x0B-ssh](./0x0B-ssh) | This project aims to deepen understanding of SSH (Secure Shell) for secure communication. It covers creating and using SSH RSA key pairs, configuring SSH clients, and automating tasks with Puppet. |
| [0x0C-web_server](./0x0C-web_server) | Involves setting up and configuring a web server with tasks like file transfer, installing Nginx, configuring DNS, managing redirections, creating custom error pages, and automating server configuration using Puppet. Focuses on understanding web servers, DNS, HTTP requests, and automation tools. |
| [0x0D-web_stack_debugging_0](./0x0D-web_stack_debugging_0) | Debugging web stack issues in a Docker container to run Apache and serve a specific page. |
| [0x0E-web_stack_debugging_1](./0x0E-web_stack_debugging_1) | Advanced debugging techniques for web stacks, focusing on diagnosing and fixing Nginx configurations to ensure it runs properly on port 80. |
| [0x0F-load_balancer](./0x0F-load_balancer) | Implementing load balancers for high availability, using HAProxy to manage multiple web servers and automate configurations with custom HTTP headers. |
| [0x10-https_ssl](./0x10-https_ssl) | Setting up SSL for secure web communications, including configuring HAproxy for SSL termination and HTTP to HTTPS redirection. |
| [0x11-what_happens_when_you_type_google_com_in_your_browser_and_press_enter](./0x11-what_happens_when_you_type_google_com_in_your_browser_and_press_enter) | Understanding the processes that occur when accessing a website. |
| [0x12-web_stack_debugging_2](./0x12-web_stack_debugging_2) | This project focuses on debugging and configuring a web stack to enhance security and functionality, including running processes as different users and configuring Nginx to run under specific conditions. |
| [0x13-firewall](./0x13-firewall) | This project focuses on configuring and managing firewall rules using UFW on a Linux server to enhance security by blocking unwanted traffic and implementing port forwarding.  |
| [0x14-mysql](./0x14-mysql) |This project involves setting up and configuring MySQL replication between primary and replica servers, creating and managing MySQL backups, and enhancing database administration skills. |
| [0x15-api](./0x15-api) | This project focuses on gathering data from a REST API and exporting it in various formats such as CSV and JSON, enhancing skills in API interaction and data handling in Python. |
| [0x16-api_advanced](./0x16-api_advanced) | This project involves creating Python scripts to interact with the Reddit API, focusing on tasks like querying subscriber counts, retrieving hot posts, and handling pagination, enhancing skills in API interactions and data parsing.  |
| [0x17-web_stack_debugging_3](./0x17-web_stack_debugging_3) | This project focuses on debugging an Apache server returning a 500 Internal Server Error using strace, applying a fix, and automating the solution with Puppet, enhancing skills in server management and debugging. |
| [0x18-webstack_monitoring](./0x18-webstack_monitoring) | This project focuses on setting up monitoring for a web stack using Datadog, covering installation of the Datadog agent, monitoring metrics, creating dashboards, and configuring alerts for effective infrastructure management. |
| [0x19-postmortem](./0x19-postmortem) | This report analyzes a Sev-2 incident where the Apache server in a Docker container failed, leading to a 2-hour outage of the "Hello Holberton" page. It includes a timeline of events, root cause analysis, actions taken for resolution, and recommendations for preventing future occurrences. |
| [0x1A-application_server](./0x1A-application_server) |  |
| [0x1B-web_stack_debugging_4](./0x1B-web_stack_debugging_4) | This project focuses on debugging an Nginx web server to improve performance under load and adjust OS configurations for user limits. It involves using Puppet manifests to automate configurations, with tasks centered on enabling the server to handle 2000 requests and modifying user limits to prevent "Too many open files" errors. |
| [attack_is_the_best_defense](./attack_is_the_best_defense) | | This project focuses on debugging an Nginx web server to improve performance under load and adjust OS configurations for user limits. It involves using Puppet manifests to automate configurations, with tasks centered on enabling the server to handle 2000 requests and modifying user limits to prevent "Too many open files" errors. |
| [command_line_for_the_win](./command_line_for_the_win) | This project focuses on enhancing Bash command line skills through a series of tasks involving scripting, loops, conditionals, and file operations. Participants complete challenges on CMD CHALLENGE, take screenshots of their progress, and upload these to a sandbox environment using SFTP. The project emphasizes practical command line techniques and task automation.  |

This repository serves as a comprehensive resource for anyone looking to enhance their skills in system engineering and DevOps practices.
